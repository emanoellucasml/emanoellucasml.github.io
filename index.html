Entendendo variáveis, escopo e içamento em JavaScript

Introdução

Variáveis são uma parte fundamental de muitas linguagens de programação, e entre os primeiros e mais importantes conceitos para novatos aprenderem. Há um número de diferentes propriedades de variáveis em JavaScript, assim como muitas regras que precisam ser seguidas ao nomeá-las. Em JavaScript, há três palavras-chave usadas para declarar uma variável. var, let, e const — e cada uma afeta como o código vai interpretar a variável diferentemente. .var, let, and const — and each one affects how the code will interpret the variable differently.

Esse tutorial vai cobrir o que variáveis são, como declará-las e nomeá-las, e também dar uma olhada mais de perto na diferença entre var, let, and const. Nós também revisaremos os efeitos de içamento e a significância de escopo global e local para o comportamento de uma variável.

Entendendo Variáveis

Uma variável é um recipiente nomeado utilizado para armazenar valores. Uma informação que podemos referir várias vezes pode ser armazenada em uma variável para uso ou modificação posterior. Em JavaScript, o valor contido em uma variável pode ser qualquer tipo de dado JavaScript, incluindo um número, string, ou objeto.

Antes da especificaçãoECMAScript 2015 (ES6) na qual JavaScript hoje é baseado, havia apenas um jeito de declarar variáveis — usando a palavra-chave var. Como resultado, a maioria dos códigos e fontes de aprendizado mais antigos usarão apenas var para variáveis. Nós veremos a diferença entre as palavras-chavevar, let, e const na seção delas abaixo.

Podemos usar var para demonstrar o conceito de uma variável em si. No exemplo abaixo nós vamosdeclarar uma variável, e atribuir o valor para a mesma.

// Atribui o valor (string) "Sammy" para o identificador "usename"
var username = "sammy_shark";

Essa declaração consiste de algumas partes:

  • A declaração de uma variável usando a palavra-chave var
  • O nome da variável (ou identificador), username
  • A operação de atribuição, representada pelo operador =
  • O valor sendo atribuído, "sammy_shark"

Agora podemos usar username no código. JavaScript vai lembrar queusername representa o string value sammy_shark.

// checa se a variável é igual ao valor
if (username === "sammy_shark") {
  console.log(true);
}

Saída
true

Como mencionado anteriormente, variáveis podem ser usadas para representar qualquer tipo de dado JavaScript. Nesse exemplo, declararemos variáveis com valores number, object, Boolean, e null.

// Declaração de várias variáveis
var name = "Sammy";
var spartans = 300;
var kingdoms = [ "mammals", "birds", "fish" ];
var poem = { roses: "red", violets: "blue" }; 
var success = true;
var nothing = null;

Usando console.log, podemos ver o valor contido em uma variável específica.

// envia a variável spartans para o console
console.log(spartans);

Saída
300

Variáveis armazenam dados na memória que podem ser acessados e modificados mais tarde. Variáveis também podem ser reatribuídas e a elas dado um novo valor. O exemplo simplificado abaixo demonstra como uma senha pode ser armazenada em uma variável e então atualizada.

// Atribui valor à variável password/span>
var password = "hunter2";

// Reatribui o valor da variável com um novo valor.
password = "hunter3";

console.log(password);

Output
'hunter3'

Num programa real, uma variável muito provavelmente seria armazenada de forma segura em um banco de dados. Esse exemplo, no entanto, ilustra uma situação na qual podemos precisar atualizar o valor de uma variável. O valor de password era hunter2, mas o reatribuímos para hunter3 que é o valor que JavaScrit reconhece desse ponto em diante.

Nomeando variáveis

Nomes de variáveis são conhecidos como identificadores Em JavaScript. Muitas regras para identificadores são discutidas em Entendendo a sintaxe e estrutura do código em JS, resumidas aqui:

  • Variable names can consist only of letters (a-z), numbers (0-9), dollar sign symbols ($), and underscores (_)
  • Variable names cannot contain any whitespace characters (tabs or spaces)
  • Numbers cannot begin the name of any variable
  • There are several reserved keywords which cannot be used as the name of a variable
  • Variable names are case sensitive

JavaScript também tem a conveção de usar camel case(sometimes stylized as camelCase) nos nomes de funções e variáveis declaradas com var or let. This is the practice of writing the first word lowercase, e então ir escrevendo maiúsculo a primeira letra de cada palvra subsequente sem espaços entre elas. A maioria das variáveis que não são constantes vão seguir essa convenção, com algumas exceções. The names of variables que são constantes, declaradas com a palavra-chave const keyword, são tipicamente escritas todas em letras maiúsculas.

Isso pode parecer muitas regras para aprender, mas, rapidamente, torna-se-á natural escrever nomes de variáveis válidos e convencionais.

Diferença entrevar, let, e const

JavaScript tem três palavras-chave diferentes para declarar uma variável, que adiciona uma camada extra de complexidade à linguagem. As diferenças entre as três são baseadas no escopo, içamento e reatribuição.

Palavra-chave Escopo Içamento Permite reatribuição Permite redeclaração
var Escopo de função sim sim sim
let Escopo de bloco não sim não
const Escopo de bloco não não não

Você pode estar imaginando qual dos três você deve usar em seus scripts. Uma pŕatica comumente aceita é usar const sempre que possível, e let no caso de loops e reatribuição. Geralmente, var pode ser evitado fora do trabalho com códigos legados.

Escopo de variáveis

Escopo em Javascript refere-se ao contexto de código atual, que determina a acessibilidade de variáveis para o JavaScript. Os dois tipos de escopos são local e global:

  • Variáveis globais são aquelas declaradas fora de um bloco
  • Variáveis locais são aquelas declaradas dentro de um bloco

No exemplo abaixo, criaremos uma variável global.

// Inicializa a variável global
var creature = "wolf";

Nós aprendemos que variáveis podem ser reatribuídas. Usando escopo loca, podemos na verdade criar novas variáveis com o mesmo nome que a variável num escopo externo sem mudar o reatribuir o valor original.

No exemplo abaixo, criaremos uma variável global de espécies.Dentro da função é uma variável local com o mesmo nome. Enviando-as para o console, podemos ver como o valor da variável é diferente dependendo do escopo, e o valor original não é alterado.

// Inicializa a variável global
var species = "human";

function transform() {
  // Inicializa uma variável local com função de escopo
  var species = "werewolf";
  console.log(species);
}

// Mostra as variáveis local e global
console.log(species); 
transform();
console.log(species);

Output
human
werewolf
human

Nesse exemplo, a variável local tem escopo de função. Variáveis declaradas com a palavra-chave var têm sempre escopo de função, o que significa que elas reconhecem funções como tendo um escopo separado. Essa variável de escopo local é, por isso, inacessível no escopo global.

As novas palavras-chavelet and const, contanto, tem escopo de bloco. Isso significa que um novo escopo local é criado a partir de qualquer tipo de bloco, incluindo blocos de funções, if declerações, e loops for e while

Para ilustrar a diferença entre variáveis que têm escopo de bloco e variáveis que têm escopo de função, vamos atribuir uma nova variável em um bloco if usando let.

var luaCheia = true;

// Inicializa a variável global
let especies = "humano";

if (luaCheia) {
  // Inicializa a variável de escopo de bloco
  let especies = "lobisomen";
  console.log(`É lua cheia. O Lupin atualmente é um ${especies}.`);
}

console.log(`Não é lua cheia. O Lupin atualmente é um ${species}.`);

Saída
É lua cheia. Lupin é atualmente um lobisomen.
Não é lua cheia. Lupin é atualmente um humano.

Nesse exemplo, a variável especies tem apenas um valor global (human), e outro valor local(werewolf). Se fóssemos usar var, no entanto, haveria um resultado diferente.

// Usando var para inicializar uma variável
var especies = "humano";

if (luaCheia) {
  // Tentativa de criar uma nova variável num bloco
  var especies = "lobisomen";
  console.log(`É lua cheia. O Lupin atualmente é um ${especies}.`);
}

console.log(`Não é lua cheia. Lupin é atualmente um ${species}.`);

Saída
É lua cheia. Lupin é atualmente um lobisomen.
É lua cheia. Lupin é atualmente um lobisomen.

No resultado desse exemplo, ambas as variáveis - global e de escopo - terminaram com o mesmo valor lobisomen. Isso é porque, em vez de criar uma nova variável local com var, você está reatribuindo a mesma variável no mesmo escopo. var não reconhece if como parte de um novo e diferente escopo. Geralmente é recomendado que você declare variáveis que são block-scoped (de escopo de bloco), pois produzem códigos que têm menos probabilidade de sobrepor-se a valores de variáveis não intencionalmente.

Içamento (hoisting)

Na maioria dos exemplos até agora, usamos var para declarar uma variável, e a inicializamos com um valor. Depois de declarar e inicializar, podemos acessar o reatribuir a variável.

Se tentarmos usar a variavel antes dessa ser declarada e inicializada, obteremos um undefined.

// Tentativa de usar uma variável antes de declaralá-la
console.log(x);

// Atribuição da variável
var x = 100;

Saída
undefined

Contudo, se omitirmos a palavra-chave var, não estamos mais declarando a variável, apenas inicializando-a. Obteremos um ReferenceError e a execução do script é interrompida

//Tentativa de usar a variável antes de declará-la
console.log(x);

// Atribuição da variável sem var
x = 100;

Output
ReferenceError: x is not defined

A razão para isso é devido ao hoisting, um comportamento do JavaScript no qual declarações de variáveis funções são movidas para o topo do escopo. Uma vez que apenas a declaração é içada, não a inicialização, o valor no primeiro exemplo retornaundefined.

Para demonstrar este conceito mais claramente, abaixo está o código que escrevemos e como o JavaScript realmente o interpretou.

// O código que escrevemos
console.log(x);
var x = 100;

// Como o JavaScript o interpretpi
var x;
console.log(x);
x = 100;

O JavaScript salvoux na memória como variável antes da execução do script. Como foi chamado ainda antes de ser definido , o resultado éundefined e não 100. Mas um ReferenceError não aparece e interrompe a execução do script. Embora a palavra-chave varvar não mudou realmente a localização do var, essa é uma representação valiosa de como o içamento funciona. Esse comportamento pode causar erros, porém, porque o programador que escreveu esse código provavelmente espera que a saída dex seja true, quando é, em vez disso, undefined.

Também podemos ver, no próximo exemplo, como o içamento lida com resultados inesperados:

// Inicializa x no escopo global
var x = 100;

function hoist() {
  // Uma condição que não deve afectar o resultado do código
  if (false) {
    var x = 200;
  }
  console.log(x);
}

hoist();

Saída
undefined

Nesse exemplo, declaramos x para ser 100 globalmente. Dependendo de uma declaração if, x poderia mudar para 200, mas já que a condição foi false não deveria ter afetado o valor de x. Em vez disso, x foi içado para o topo da fuunção hoist() , e o valor se tornou undefined.

Este tipo de comportamento imprevisível pode potencialmente causar bugs em um programa. Como let e const são block-scoped, não serão içados dessa forma, como se vê abaixo.

// Inicializa x no escopo global
let x = true;

function hoist() {
  // Inicializa x no escopo de função
  if (3 === 4) {
    let x = false;
  }
  console.log(x);
}

hoist();

Saída
true

Declaração duplicada de variáveis, o que é possível com var,lançará um erro com let and const.

// Tentativa de sobrescrever uma variável declarada com var
var x = 1;
var x = 2;

console.log(x);

Output
2

// Attempt to overwrite a variable declared with let
let y = 1;
let y = 2;

console.log(y);

Saída
Uncaught SyntaxError: Identifier 'y' has already been declared

Para resumir, variáveis introduzidas com var tem o potencial de serem afetadas por hoisting, um mecanismo em JavaScript no qual as declarações de variáveis são guardadas na memória. Isto pode resultar em variáveis indefinidas no código. A introdução delet and const resolve este problema lançando um erro ao tentar usar uma variável antes de declará-la ou ao tentar declarar uma variável mais de uma vez.

Constantes

Muitas linguagens de programação apresentam constants, que são valores que não podem ser alterados. Em JavaScript, o identificador const é modelado após constantes, e os valores atribuídos a const não pode ser reatribuído.

É comum a convenção de escrever todos os identificadores const em letra maiúscula. Isto os deixa facilmente distinguíveis de outros valores

No exemplo abaixo, inicializamos a variável ESPECIES como uma constante com a palavra-chave const. Tentar reatribuir a variável resultará em um erro

// Atribui valor a const
const ESPECIES = "humano"; 

// Tentativa de reatribuir valor
ESPECIES = "lobisomen";

console.log(SPECIES);

Saída
Uncaught TypeError: Assignment to constant variable.

Já que valores const não podem ser reatribuídos, eles precisam ser declarados e inicializados ao mesmo tempo, ou também irão lançar um erro.

//Declara mas não inicializa uma constante
const TODO;

console.log(TODO);

Saída
Uncaught SyntaxError: Missing initializer in const declaration

Os valores que não podem mudar na programação são conhecidos como imutáveis, enquanto os valores que podem ser alterados são mutáveis . Embora const valores não podem ser reatribuídos, eles são mutáveis, pois é possível modificar as propriedades dos objetos declarados comconst.

// Cria um carro com duas propriedades
const CARRO = {
    cor: "azul",
    preço: 15000
}

// Modifica uma propriedade de CARRO
CARRO.preço = 20000;

console.log(CARRO);

Saída
{ cor: 'azul', preco: 20000 }

As constantes são úteis para deixar claro para si mesmo e a outros programadores que trabalham em um projeto com você que a variável pretendida não deve ser reatribuída. Se você espera que uma variável possa ser modificada no futuro, você provavelmente vai querer usar, em vez disso, let para declarar a variável.

Conclusão

Neste tutorial, nós revisamos o que é uma variável, as regras de nomeação de uma variável e como reatribuir valores de variáveis. Também aprendemos sobre escopo e levantamento, algumas das limitações da palavra-reservada var original, assim como let e const retificam esses problemas.
